# Extract feedback — run in an active project Claude session

Paste this into a live Claude Code / Claude.ai session that has history with you. The instance will mine its transcript / memory for preferences and corrections you haven't written down yet. Fold the output back into your personal CLAUDE.md.

---

Pull every correction, preference, and "this is how they work" signal you can find in this session's transcript (and if you have access, the last ~20 sessions with me). I want five outputs, in this order:

## 1. Raw observations
Unfiltered "this is how I work" list. Things I say, patterns I push back on, shortcuts I prefer, fears I've expressed, phrasings I repeat, how I react to misses, register shifts. Don't filter for "worth saving" — I'll prune. Group loosely: tone, process, decision style, aesthetic / craft, builder mindset, fears, phrasings I repeat, workflow tics, relationship.

## 2. Feedback not yet captured in memory
Go through my `memory/feedback_*.md` and `memory/MEMORY.md` files. What corrections or confirmations are you running on that aren't in there? Call each one out — a one-line rule + why + how to apply.

## 3. One-page onboarding README for a fresh Claude
If you had to hand a brand-new Claude instance a single page on working with me — no codebase context — what would it say? Sections: language, non-negotiables, how I think, process, vibe.

## 4. Generalized rules from corrections (last ~20 sessions)
List every moment I corrected your approach. Generalize each into a rule. Format: `R<n>. <rule>. **Why:** <one line from the specific incident>. **How to apply:** <concrete guidance>`. Don't list corrections raw — generalize.

## 5. Delegation mismatch
Two columns:
- **Things I ask you to do that I should keep doing myself** (e.g. aesthetic judgment, writing posts in my voice).
- **Things I do manually that I should be delegating to you** (e.g. cross-referencing prod logs, drafting commit messages when asked).

---

Density > completeness. Dense bullets > long paragraphs. No preamble, no recap.
